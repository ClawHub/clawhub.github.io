Spring-IOC加载Bean【十七】简单总结
发表于 2019-08-09| 更新于 2020-02-24|spring-ioc
字数总计: 1.9k|阅读时长: 7 分钟| 阅读量:

Spring加载Bean确实很复杂，估计要看几遍才能摸清楚脉络，分析这一块源码大大小小耗费了好多天，终于耐着性子简单的分析完了。
写这个总结还得拉出一段代码：

即最后一行bean的加载逻辑，通过DefaultListableBeanFactory的getBean()方法，
最终调用到AbstractBeanFactory的doGetBean(final String name, @Nullable final Class requiredType,@Nullable final Object[] args, boolean typeCheckOnly)主流程。

  1. 从缓存中获取原始bean实例
  2. 创建原始bean实例
    1. 获取BeanWrepper
    2. 属性填充
    3. 初始化bean
  3. 从原始bean实例中获取我们需要的对象

1、从缓存中获取原始bean实例

getSingleton(String beanName),只有单例模式才会从缓存中获取原始bean实例。

2、创建原始bean实例

如果缓存没有，也没有parentBeanFactory，则调用createBean(beanName, mbd, args)方法获取，最终会调用doCreateBean(final String beanName, final RootBeanDefinition mbd, final @Nullable Object[] args)方法。
这就是巨无霸了！！方法主要分了三个大部分：首先通过createBeanInstance(beanName, mbd, args)方法，获取BeanWrepper；之后将属性填充到BeanWrapper,最后初始化bean。
具体源码分析可以参考系列文章。

3、从原始bean实例中获取我们需要的对象

获取的原始bean对象是不能直接返回的，需要最后通过getObjectForBeanInstance(Object beanInstance, String name, String beanName, @Nullable RootBeanDefinition mbd)
方法处理一下，这个方法我在以前也分析过了。

感慨一下，Spring加载Bean的逻辑代码的体量真的好大，来来回回看了好久，总算结束，虽然对其内部还是一知半解，但是明显的比以前的认知好多了，总的来说收获很大，记住要静下心来，硬看。
